# Security policy

## Supported versions

Only the latest release on the `main` branch receives security fixes.

## Reporting a vulnerability

Do not open a public issue for a suspected vulnerability, and never include a Substack session token, cookie, or publication-private data in a report.

Use [GitHub's private security advisory form](https://github.com/cucoleadan/substack-edge-sdk/security/advisories/new) to report a vulnerability. Include a clear description, reproduction steps, impact, and a minimal redacted proof of concept.

Maintainers will acknowledge reports within seven days and coordinate a fix and disclosure timeline with the reporter.
